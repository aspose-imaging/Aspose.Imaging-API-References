## DibBitCount enumeration

L'énumération BitCount spécifie le nombre de bits qui définissent chaque pixel et le nombre maximal de couleurs dans un bitmap indépendant du périphérique (DIB).

```csharp
public enum DibBitCount : short
```

### Valeurs

| Nom | Évaluer | La description |
| --- | --- | --- |
| Bitcount0 | `0` | Le nombre de bits par pixel n'est pas défini. L'image DEVRAIT être au format JPEG ou PNG. Aucun de ces formats n'inclut de table de couleurs, donc cette valeur spécifie qu'aucune table de couleurs n'est présente. Voir [JFIF] et [RFC2083] pour plus d'informations concernant les formats de compression JPEG et PNG. |
| Bitcount1 | `1` | L'image est spécifiée avec deux couleurs. Chaque pixel du bitmap est représenté par un seul bit. Si le bit est clair, le pixel est affiché avec la couleur de la première entrée de la table des couleurs ; si le bit est activé, le pixel a la couleur de la deuxième entrée de la table. |
| Bitcount2 | `4` | L'image est spécifiée avec un maximum de 16 couleurs. Chaque pixel du bitmap est représenté par un index de 4 bits dans la table des couleurs , et chaque octet contient 2 pixels. |
| Bitcount3 | `8` | L'image est spécifiée avec un maximum de 256 couleurs. Chaque pixel du bitmap est représenté par un index de 8 bits dans la table de couleurs , et chaque octet contient 1 pixel. |
| Bitcount4 | `16` | L'image est spécifiée avec un maximum de 2^16 couleurs. Chaque pixel du bitmap est représenté par une valeur de 16 bits |
| Bitcount5 | `24` | Le bitmap a un maximum de 2^24 couleurs et le champ Couleurs de DIB est NULL. Chaque triplet de 3 octets dans le tableau bitmap représente les intensités relatives de bleu, vert et rouge, respectivement, pour un pixel. La table de couleurs Colors est utilisée pour optimiser les couleurs utilisées sur les appareils basés sur une palette et DOIT contenir le nombre d'entrées spécifiées par le champ ColorUsed de BitmapInfoHeader Object |
| Bitcount6 | `32` | Le bitmap a un maximum de 2^24 couleurs |
