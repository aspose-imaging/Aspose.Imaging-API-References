## EmfPlusPathPointFlags enumeration

Un entier non signé 32 bits qui spécifie comment interpréter les points et les types de points associés définis par cet objet. C (1 bit) : s'il est défini, le tableau PathPoints spécifie les emplacements absolus dans l'espace de coordonnées avec un entier 16 bits coordinates. S'il est clair, le tableau PathPoints spécifie des emplacements absolus dans l'espace de coordonnées avec des coordonnées à virgule flottante de 32 bits. Remarque Si le drapeau P (ci-dessous) est défini, ce drapeau PEUT être clair et DOIT être ignoré. R ( 1 bit) : S'ils sont définis, les types de points dans le tableau PathPointTypes sont spécifiés par les objets EmfPlusPathPointTypeRle (section 2.2.2.32), qui utilisent la compression RLE (run-length encoding) et/ou les objets EmfPlusPathPointType (section 2.2.2.31). Voir [MS-WMF] section 3.1.6 pour plus d'informations sur la compression RLE. Si désactivé, les types de points dans le tableau PathPointTypes sont spécifiés par les objets EmfPlusPathPointType. P (1 bit) : si défini, chaque élément du tableau PathPoints spécifie un emplacement dans l'espace de coordonnées relatif à l'emplacement spécifié par l'élément précédent dans le tableau. Dans le cas du premier élément de PathPoints, un emplacement précédent aux coordonnées (0,0) est supposé. Si clair, chaque élément du tableau PathPoints spécifie un emplacement absolu.
