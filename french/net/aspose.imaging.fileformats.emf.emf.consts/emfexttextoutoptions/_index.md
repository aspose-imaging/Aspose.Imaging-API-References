## EmfExtTextOutOptions enumeration

L'énumération ExtTextOutOptions spécifie les paramètres qui contrôlent divers aspects de la sortie de texte par les enregistrements EMR_SMALLTEXTOUT(section 2.3.5.37) et dans les objets EmrText.

```csharp
[Flags]
public enum EmfExtTextOutOptions
```

### Valeurs

| Nom | Évaluer | La description |
| --- | --- | --- |
| ETO_OPAQUE | `2` | Ce bit indique que la couleur d'arrière-plan actuelle DEVRAIT être utilisée pour remplir le rectangle |
| ETO_CLIPPED | `4` | Ce bit indique que le texte DEVRAIT être découpé dans le rectangle. |
| ETO_GLYPH_INDEX | `10` | Ce bit indique que les codes des caractères dans une chaîne de texte de sortie sont en fait des index des glyphes de caractères dans une police TrueType. Les index glyphes sont spécifiques à la police, donc pour afficher les caractères corrects lors de la lecture, la police utilisée DOIT être identique à la police utilisée pour générer les index. |
| ETO_RTLREADING | `80` | Ce bit indique que le texte DOIT être disposé dans l'ordre de lecture de droite à gauche, au lieu de l'ordre de gauche à droite par défaut. Cela DEVRAIT être appliqué uniquement lorsque la font sélectionnée dans le contexte du périphérique de lecture est soit en hébreu, soit en arabe |
| ETO_NO_RECT | `100` | Ce bit indique que l'enregistrement ne spécifie pas de rectangle de délimitation pour la sortie texte. |
| ETO_SMALL_CHARS | `200` | Ce bit indique que les codes des caractères dans une chaîne de texte de sortie sont de 8 bits, dérivés des octets de poids faible des codes de caractères Unicode UTF16-LE 16 bits, dans lesquels l'octet de poids fort est supposé être 0. |
| ETO_NUMERICSLOCAL | `400` | Ce bit indique que pour afficher des nombres, des chiffres appropriés aux paramètres régionaux DEVRAIENT être utilisés |
| ETO_NUMERICSLATIN | `800` | Ce bit indique que pour afficher des nombres, les chiffres européens DEVRAIENT être utilisés |
| ETO_IGNORELANGUAGE | `1000` | Ce bit indique qu'aucun traitement spécial du système d'exploitation pour le placement des glyphes ne doit être effectué sur les chaînes de droite à gauche ; c'est-à-dire que tout le positionnement des glyphes DEVRAIT être pris en charge par le dessin et les enregistrements d'état dans le métafichier |
| ETO_PDY | `2000` | Ce bit indique que les valeurs de déplacement horizontal et vertical des caractères DEVRAIENT être fournies |
| ETO_REVERSE_INDEX_MAP | `10000` | Ce bit est réservé et NE DOIT PAS être utilisé |
