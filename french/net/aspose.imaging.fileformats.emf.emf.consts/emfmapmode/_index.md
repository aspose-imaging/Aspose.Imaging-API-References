## EmfMapMode enumeration

L'énumération MapMode est utilisée pour définir l'unité de mesure pour transformer les unités d'espace de page en unités d'espace de périphérique et pour définir l'orientation des axes de dessin.

```csharp
public enum EmfMapMode
```

### Valeurs

| Nom | Évaluer | La description |
| --- | --- | --- |
| MM_TEXT | `1` | Chaque unité logique est mappée à un pixel de périphérique. Le x positif est à droite ; y positif est down. |
| MM_LOMETRIC | `2` | Chaque unité logique est mappée à 0,1 millimètre. Le x positif est à droite ; y positif est en hausse. |
| MM_HIMETRIC | `3` | Chaque unité logique est mappée à 0,01 millimètre. Le x positif est à droite ; y positif est en hausse. |
| MM_LOENGLISH | `4` | Chaque unité logique est mappée à 0,01 pouce. Le x positif est à droite ; y positif est up |
| MM_HIENGLISH | `5` | Chaque unité logique est mappée à 0,001 pouce. Le x positif est à droite ; y positif est en hausse. |
| MM_TWIPS | `6` | Chaque unité logique est mappée à un vingtième du point d'une imprimante (1/1440 pouce, également appelé "twip"). Le x positif est à droite ; y positif est en hausse. |
| MM_ISOTROPIC | `7` | Les unités logiques sont mappées sur des unités arbitraires avec des axes à échelle égale ; c'est-à-dire qu'une unité le long de l'axe des x est égale à une unité le long de l'axe des y. Les enregistrements EMR_SETWINDOWEXTEX et EMR_SETVIEWPORTEXTEX DEVRAIENT être utilisés pour spécifier les unités et l'orientation des axes. Des ajustements DOIVENT être effectués si nécessaire pour s'assurer que les unités x et y restent de la même taille. Par exemple, lorsque l'étendue de la fenêtre est définie, la fenêtre DOIT être ajustée pour conserver les unités isotropes. |
| MM_ANISOTROPIC | `8` | Les unités logiques sont mappées sur des unités arbitraires avec des axes arbitrairement mis à l'échelle. Les enregistrements EMR_SETWINDOWEXTEX et EMR_SETVIEWPORTEXTEX DOIVENT être utilisés pour spécifier les unités, l'orientation et la mise à l'échelle. |
