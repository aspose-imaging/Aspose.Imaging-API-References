## EmfPlusStringFormatFlags enumeration

StringFormat 标志指定图形文本布局的选项，包括方向、剪切和字体处理。这些标志可以组合起来指定多个选项。

```csharp
[Flags]
public enum EmfPlusStringFormatFlags : uint
```

### 价值观

| 姓名 | 价值 | 描述 |
| --- | --- | --- |
| StringFormatDirectionRightToLeft | `1` | 如果设置，字符串的阅读顺序应该是从右到左。对于水平文本，这意味着从右到左读取字符。对于垂直文本，这意味着从右到左读取列。 如果清晰，水平或垂直文本应该从左到右阅读。 |
| StringFormatDirectionVertical | `2` | 如果设置，单独的文本行应该在显示设备上垂直绘制。 如果清除，应水平绘制单独的文本行，每一行都在前一行下方。 |
| StringFormatNoFitBlackBox | `4` | 如果设置，必须允许部分字符悬垂在文本布局矩形上。 如果清除，悬垂文本布局矩形边界的字符必须重新定位以避免悬垂。 斜体，“f”是可以具有悬垂部分的字符的示例。 |
| StringFormatDisplayFormatControl | `20` | 如果设置，控制字符应该作为代表 Unicode 字形出现在输出中。 |
| StringFormatNoFontFallback | `400` | 如果设置，则应为请求字体中不支持的字符使用替代字体。 如果清除，请求字体中缺少的字符应该显示为“字体缺失”字符，它可能是一个空心方块。 |
| StringFormatMeasureTrailingSpaces | `800` | 如果设置，每行末尾的空格必须包含在字符串长度的测量中。 如果清除，每行末尾的空格必须从字符串长度的测量中排除。 |
| StringFormatNoWrap | `1000` | 如果设置，则超出文本布局矩形末尾的字符串不得换行到下一行。 如果清除，超出文本布局矩形末尾的字符串必须在边界矩形内的最后一个单词边界处断开，并且字符串的其余部分必须换行到下一行。 |
| StringFormatLineLimit | `2000` | 如果设置，整行文本应该被输出并且不应该被字符串的布局矩形剪裁。 如果清除，文本布局应该继续，直到所有行都被输出，或者直到其他行由于剪裁而变得不可见。 此标志可用于拒绝或允许文本行被不是行高倍数的布局矩形部分遮挡。为了使所有文本可见，布局矩形至少与一行的高度一样高。 |
| StringFormatNoClip | `4000` | 如果设置，则应允许显示超出字符串布局矩形的文本。 如果清除，所有延伸到布局矩形之外的文本都应该被剪裁。 |
| StringFormatBypassGdi | `80000000` | 此标志可用于指定特定于实现的过程以呈现文本。 |
