## EmfPlusStringFormatFlags enumeration

StringFormat-flaggorna anger alternativ för grafiktextlayout, inklusive riktning, klippning och teckensnittshantering. Dessa flaggor kan kombineras för att ange flera alternativ.

```csharp
[Flags]
public enum EmfPlusStringFormatFlags : uint
```

### Värderingar

| namn | Värde | Beskrivning |
| --- | --- | --- |
| StringFormatDirectionRightToLeft | `1` | Om inställt, SKA läsordningen för strängen vara höger till vänster. För horisontell text betyder det att tecken läses från höger till vänster. För vertikal text betyder det att kolumner läses från höger till vänster. Om tydlig, SKA horisontell eller vertikal text läsas från vänster till höger. |
| StringFormatDirectionVertical | `2` | Om det är inställt SKA enskilda textrader ritas vertikalt på displayenheten. Om de är klara SKA enskilda textrader ritas horisontellt, med varje ny rad under föregående rad. |
| StringFormatNoFitBlackBox | `4` | Om inställt, MÅSTE delar av tecken tillåtas att överhänga textlayoutrektangeln. Om det är klart, MÅSTE tecken som överhänger gränserna för textlayoutrektangeln flyttas om för att undvika överhäng. En kursiv stil, "f" är ett exempel på en tecken som kan ha överhängande delar. |
| StringFormatDisplayFormatControl | `20` | Om inställt SKA kontrolltecken visas i utdata som representativa Unicode-glyfer. |
| StringFormatNoFontFallback | `400` | Om det är inställt SKA ett alternativt teckensnitt användas för tecken som inte stöds i det efterfrågade teckensnittet. Om det är klart, SKA ett tecken som saknas i det begärda teckensnittet visas som ett tecken som "tecken saknas", vilket KAN vara en öppen ruta. |
| StringFormatMeasureTrailingSpaces | `800` | Om inställt, MÅSTE utrymmet i slutet av varje rad inkluderas i måtten på stränglängd. Om det är klart, MÅSTE utrymmet i slutet av varje rad exkluderas från måtten på stränglängd. |
| StringFormatNoWrap | `1000` | Om angivet, FÅR en sträng som sträcker sig förbi slutet av textlayoutrektangeln INTE lindas till nästa rad. Om den är ren, MÅSTE en sträng som sträcker sig förbi slutet av textlayoutrektangeln brytas vid sista ordgränsen inom den avgränsande rektangeln och resten av strängen MÅSTE lindas till nästa rad. |
| StringFormatLineLimit | `2000` | Om inställt, SKA hela rader med text matas ut och SKA INTE klippas av strängens layoutrektangel. Om den är ren, SKA textlayouten fortsätta tills alla rader matas ut, eller tills ytterligare rader inte skulle vara synliga som ett resultat av klippning. Den här flaggan kan användas antingen för att förneka eller tillåta en textrad att delvis skymmas av en layoutrektangel som inte är en multipel av radhöjden. För att all text ska vara synlig, en layoutrektangel som är minst lika hög som höjden på en rad. |
| StringFormatNoClip | `4000` | Om inställt, SKA text som sträcker sig utanför stränglayoutrektangeln tillåtas visas. Om den är ren, SKA all text som sträcker sig utanför layoutrektangeln klippas. |
| StringFormatBypassGdi | `80000000` | Den här flaggan KAN användas för att specificera en implementeringsspecifik process för rendering av text. |
