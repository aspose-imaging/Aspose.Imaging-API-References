## EmfExtTextOutOptions enumeration

ExtTextOutOptions-uppräkningen anger parametrar som styr olika aspekter av -utmatningen av text av EMR_SMALLTEXTOUT(avsnitt 2.3.5.37)-poster och i EmrText-objekt.

```csharp
[Flags]
public enum EmfExtTextOutOptions
```

### Värderingar

| namn | Värde | Beskrivning |
| --- | --- | --- |
| ETO_OPAQUE | `2` | Denna bit indikerar att den aktuella bakgrundsfärgen SKA användas för att fylla rektangeln |
| ETO_CLIPPED | `4` | Denna bit indikerar att texten SKA klippas till rektangeln. |
| ETO_GLYPH_INDEX | `10` | Den här biten indikerar att koderna för tecken i en utgående textsträng faktiskt är index för teckenglyfer i ett TrueType-teckensnitt. Glyfindex är teckensnittsspecifika, så för att visa rätt tecken vid uppspelning MÅSTE teckensnittet som används vara identiskt med teckensnittet som används för att generera indexen. |
| ETO_RTLREADING | `80` | Denna bit indikerar att texten MÅSTE läggas ut i läsordning från höger till vänster, istället för standardordningen från vänster till höger. Detta BÖR endast tillämpas när font som valts i uppspelningsenhetskontexten är antingen hebreiska eller arabiska |
| ETO_NO_RECT | `100` | Denna bit indikerar att posten inte specificerar en begränsningsrektangel för textutmatningen. |
| ETO_SMALL_CHARS | `200` | Denna bit indikerar att koderna för tecken i en utgående textsträng är 8 bitar, härledda från de låga byten av 16-bitars Unicode UTF16-LE teckenkoder, där den höga byten antas vara 0. |
| ETO_NUMERICSLOCAL | `400` | Denna bit indikerar att för att visa siffror SKA siffror som är lämpliga för lokalen användas |

| ETO_IGNORELANGUAGE | `1000` | Den här biten indikerar att ingen speciell operativsystembearbetning för glyfplacering ska utföras på höger-till-vänster-strängar; det vill säga all glyfpositionering SKA tas om hand genom att rita och ange poster i metafilen |
| ETO_PDY | `2000` | Denna bit indikerar att både horisontella och vertikala teckenförskjutningsvärden SKA tillhandahållas |
