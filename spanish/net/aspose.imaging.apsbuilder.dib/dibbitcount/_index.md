## DibBitCount enumeration

La enumeración BitCount especifica el número de bits que definen cada píxel y el número máximo de colores en un mapa de bits independiente del dispositivo (DIB).

```csharp
public enum DibBitCount : short
```

### Valores

| Nombre | Valor | Descripción |
| --- | --- | --- |
| Bitcount0 | `0` | El número de bits por píxel no está definido. La imagen DEBE estar en formato JPEG o PNG. Ninguno de estos formatos incluye una tabla de colores, por lo que este valor especifica que no hay una tabla de colores presente. Consulte [JFIF] y [RFC2083] para obtener más información sobre los formatos de compresión JPEG y PNG. |
| Bitcount1 | `1` | La imagen se especifica con dos colores. Cada píxel del mapa de bits está representado por un solo bit. Si el bit está limpio, el píxel se muestra con el color de la primera entrada en la tabla de colores; si el bit está establecido, el píxel tiene el color de la segunda entrada en la tabla. |
| Bitcount2 | `4` | La imagen se especifica con un máximo de 16 colores. Cada píxel del mapa de bits está representado por un índice de 4 bits en la tabla de colores y cada byte contiene 2 píxeles. |
| Bitcount3 | `8` | La imagen se especifica con un máximo de 256 colores. Cada píxel del mapa de bits está representado por un índice de 8 bits en la tabla de colores y cada byte contiene 1 píxel. |
| Bitcount4 | `16` | La imagen se especifica con un máximo de 2^16 colores. Cada píxel del mapa de bits se representa con un valor de 16 bits |
| Bitcount5 | `24` | El mapa de bits tiene un máximo de 2^24 colores y el campo Colores de DIB es NULL. Cada triplete de 3 bytes en la matriz de mapa de bits representa las intensidades relativas de azul, verde y rojo, respectivamente, para un píxel. La tabla de colores Colors se utiliza para optimizar los colores utilizados en dispositivos basados en paletas y DEBE contener la cantidad de entradas especificadas por el campo ColorUsed del BitmapInfoHeader Object |
| Bitcount6 | `32` | El mapa de bits tiene un máximo de 2^24 colores |
