## EmfPlusStringFormatFlags enumeration

Los indicadores StringFormat especifican opciones para el diseño de texto de gráficos, incluida la dirección, el recorte y el manejo de fuentes. Estas banderas se pueden combinar para especificar múltiples opciones.

```csharp
[Flags]
public enum EmfPlusStringFormatFlags : uint
```

### Valores

| Nombre | Valor | Descripción |
| --- | --- | --- |
| StringFormatDirectionRightToLeft | `1` | Si se establece, el orden de lectura de la cadena DEBE ser de derecha a izquierda. Para texto horizontal, esto significa que los caracteres se leen de derecha a izquierda. Para texto vertical, esto significa que las columnas se leen de derecha a izquierda. Si es claro, el texto horizontal o vertical DEBE leerse de izquierda a derecha. |
| StringFormatDirectionVertical | `2` | Si se establece, las líneas individuales de texto DEBEN dibujarse verticalmente en el dispositivo de visualización. Si está claro, las líneas individuales de texto DEBEN dibujarse horizontalmente, con cada nueva línea debajo de la línea anterior. |
| StringFormatNoFitBlackBox | `4` | Si se establece, se DEBE permitir que partes de los caracteres sobresalgan del rectángulo de diseño de texto. Si está claro, los caracteres que sobresalgan de los límites del rectángulo de diseño de texto DEBEN reposicionarse para evitar que sobresalgan. Una cursiva, "f" es un ejemplo de una personaje que puede tener partes sobresalientes. |
| StringFormatDisplayFormatControl | `20` | Si se establece, los caracteres de control DEBERÍAN aparecer en la salida como glifos Unicode representativos. |
| StringFormatNoFontFallback | `400` | Si se establece, se DEBE usar una fuente alternativa para los caracteres que no se admiten en la fuente solicitada. Si está claro, un carácter que falta en la fuente solicitada DEBE aparecer como un carácter de "fuente faltante", que PUEDE ser un cuadrado abierto. |
| StringFormatMeasureTrailingSpaces | `800` | Si se establece, el espacio al final de cada línea DEBE incluirse en las mediciones de la longitud de la cadena. Si está claro, el espacio al final de cada línea DEBE excluirse de las mediciones de la longitud de la cadena. |
| StringFormatNoWrap | `1000` | Si se establece, una cadena que se extienda más allá del final del rectángulo de diseño de texto NO DEBE pasar a la siguiente línea. el rectángulo delimitador, y el resto de la cadena DEBE pasar a la siguiente línea. |
| StringFormatLineLimit | `2000` | Si se establece, se DEBEN generar líneas completas de texto y NO SE DEBEN recortar con el rectángulo de diseño de la cadena. Si se desactiva, el diseño del texto DEBE continuar hasta que se generen todas las líneas, o hasta que las líneas adicionales no sean visibles como resultado del recorte. Este indicador se puede usar para denegar o permitir que una línea de texto quede parcialmente oculta por un rectángulo de diseño que no es un múltiplo de la altura de la línea. Para que todo el texto sea visible, un rectángulo de diseño al menos tan alto como la altura de una línea. |
| StringFormatNoClip | `4000` | Si se establece, el texto que se extiende fuera del rectángulo de diseño de la cadena DEBE permitir que se muestre. Si está claro, todo el texto que se extiende fuera del rectángulo de diseño DEBE recortarse. |
| StringFormatBypassGdi | `80000000` | Este indicador PUEDE usarse para especificar un proceso específico de implementación para representar texto. |
