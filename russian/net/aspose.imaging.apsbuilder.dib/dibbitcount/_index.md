## DibBitCount enumeration

Перечисление BitCount указывает количество битов, определяющих каждый пиксель, и максимальное количество цветов в аппаратно-независимом растровом изображении (DIB).

```csharp
public enum DibBitCount : short
```

### Ценности

| Имя | Ценность | Описание |
| --- | --- | --- |
| Bitcount0 | `0` | Количество битов на пиксель не определено. Изображение ДОЛЖНО быть в формате JPEG или PNG. Ни один из этих форматов не включает таблицу цветов, поэтому это значение указывает, что таблица цветов отсутствует. См. [JFIF] и [RFC2083] для получения дополнительной информации о форматах сжатия JPEG и PNG. |
| Bitcount1 | `1` | Изображение задается двумя цветами. Каждый пиксель в растровом изображении представлен одним битом. Если бит сброшен, пиксель отображается с цветом первой записи в таблице цветов; если бит установлен, пиксель имеет цвет второй записи в таблице. |
| Bitcount2 | `4` | Изображение указано максимум с 16 цветами. Каждый пиксель в растровом изображении представлен 4-битным индексом в таблице цветов , и каждый байт содержит 2 пикселя. |
| Bitcount3 | `8` | Изображение указано максимум с 256 цветами. Каждый пиксель в растровом изображении представлен 8-битным индексом в таблице цветов , и каждый байт содержит 1 пиксель. |
| Bitcount4 | `16` | Изображение указывается максимум с 2^16 цветами. Каждый пиксель растрового изображения представлен 16-битным значением |
| Bitcount5 | `24` | Растровое изображение имеет максимум 2^24 цвета, а поле Colors в DIB имеет значение NULL. Каждый 3-байтовый триплет в массиве растрового изображения представляет относительную интенсивность синего, зеленого и красного для пикселя соответственно. Таблица цветов Colors используется для оптимизации цветов, используемых на устройствах с палитрой, и ДОЛЖНА содержать количество записей, указанное в поле ColorUsed объекта BitmapInfoHeader Object |
| Bitcount6 | `32` | Растровое изображение имеет максимум 2^24 цвета |
