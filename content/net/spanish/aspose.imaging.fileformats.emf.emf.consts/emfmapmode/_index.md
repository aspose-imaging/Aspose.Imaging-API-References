## EmfMapMode enumeration

La enumeración MapMode se utiliza para definir la unidad de medida para transformar las unidades de espacio de página en unidades de espacio de dispositivo y para definir la orientación de los ejes de dibujo.

```csharp
public enum EmfMapMode
```

### Valores

| Nombre | Valor | Descripción |
| --- | --- | --- |
| MM_TEXT | `1` | Cada unidad lógica se asigna a un píxel de dispositivo. La x positiva está a la derecha; y positivo está abajo. |
| MM_LOMETRIC | `2` | Cada unidad lógica se asigna a 0,1 milímetros. La x positiva está a la derecha; positivo y está arriba. |
| MM_HIMETRIC | `3` | Cada unidad lógica se asigna a 0,01 milímetros. La x positiva está a la derecha; positivo y está arriba. |
| MM_LOENGLISH | `4` | Cada unidad lógica se asigna a 0,01 pulgadas. La x positiva está a la derecha; y positivo es up |
| MM_HIENGLISH | `5` | Cada unidad lógica se asigna a 0,001 pulgadas. La x positiva está a la derecha; positivo y está arriba. |
| MM_TWIPS | `6` | Cada unidad lógica se asigna a una vigésima parte del punto de una impresora (1/1440 pulgadas, también llamado "twip"). La x positiva está a la derecha; positivo y está arriba. |
| MM_ISOTROPIC | `7` | Las unidades lógicas se asignan a unidades arbitrarias con ejes de igual escala; es decir, una unidad a lo largo del eje x es igual a una unidad a lo largo del eje y. Los registros EMR_SETWINDOWEXTEX y EMR_SETVIEWPORTEXTEX DEBEN utilizarse para especificar las unidades y la orientación de los ejes. DEBEN realizarse los ajustes necesarios para garantizar que las unidades x e y sigan siendo del mismo tamaño. Por ejemplo, cuando se establece la extensión de la ventana, la ventana gráfica DEBE ajustarse para mantener las unidades isotrópicas. |
| MM_ANISOTROPIC | `8` | Las unidades lógicas se asignan a unidades arbitrarias con ejes arbitrariamente escalados. Los registros EMR_SETWINDOWEXTEX y EMR_SETVIEWPORTEXTEX DEBERÍAN utilizarse para especificar las unidades, la orientación y la escala. |
