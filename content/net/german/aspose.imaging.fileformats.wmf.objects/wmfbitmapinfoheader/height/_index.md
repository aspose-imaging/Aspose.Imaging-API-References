## WmfBitmapInfoHeader.Height property

Ruft eine 32-Bit-Ganzzahl mit Vorzeichen ab oder legt diese fest, die die Höhe der DIB in Pixel definiert. Dieser Wert DARF NICHT Null sein. Wenn dieser Wert positiv ist, ist die DIB eine Bitmap von unten nach oben und ihr Ursprung ist die untere linke Ecke. Wenn dieser Wert negativ ist, ist die DIB eine Bitmap von oben nach unten, und sein Ursprung ist die obere linke Ecke. Top-Down-Bitmaps unterstützen keine Komprimierung. Dieses Feld SOLLTE die Höhe der dekomprimierten Bilddatei angeben, wenn der Komprimierungswert das JPEG- oder PNG -Format angibt.

public int Height { get; set; }
