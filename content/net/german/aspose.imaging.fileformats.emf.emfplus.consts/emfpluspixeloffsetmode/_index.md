## EmfPlusPixelOffsetMode enumeration

Die PixelOffsetMode-Enumeration definiert, wie Pixel versetzt werden, wodurch der Kompromiss zwischen Rendering-Geschwindigkeit und Qualität angegeben wird.

```csharp
public enum EmfPlusPixelOffsetMode : byte
```

### Werte

| Name | Wert | Beschreibung |
| --- | --- | --- |
| PixelOffsetModeDefault | `0` | Pixel sind auf ganzzahligen Koordinaten zentriert und geben Geschwindigkeit statt Qualität an. |
| PixelOffsetModeHighSpeed | `1` | Pixel werden auf ganzzahlige Koordinaten zentriert, wie bei PixelOffsetModeNone. Höhere Geschwindigkeit auf Kosten der Qualität ist vorgeschrieben. |
| PixelOffsetModeHighQuality | `2` | Pixel werden auf halbzahligen Koordinaten zentriert, wie bei PixelOffsetModeHalf. Höhere Qualität auf Kosten der Geschwindigkeit wird vorgegeben. |
| PixelOffsetModeNone | `3` | Pixel sind auf den Ursprung zentriert, was bedeutet, dass das Pixel den Bereich von -0,5 bis 0,5 sowohl auf der x- als auch auf der y-Achse abdeckt und sein Mittelpunkt bei (0,0) liegt. |
| PixelOffsetModeHalf | `4` | Pixel sind auf halbzahligen Koordinaten zentriert, was bedeutet, dass das Pixel den Bereich von 0 bis 1 sowohl auf der x- als auch auf der y-Achse abdeckt und sein Mittelpunkt bei (0,5,0,5) liegt. Durch das Versetzen von Pixeln während des Renderns kann die Renderqualität auf Kosten der Rendergeschwindigkeit verbessert werden. |
